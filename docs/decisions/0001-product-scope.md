# ADR 0001 — Product scope

**Status:** Accepted (2026-07-11)  
**Deciders:** Founder

## Context

Margin6 documentation mixed marketing claims, RestaurantIQ-era plans, and verified audit findings. Agents and contributors need a single product boundary.

## Decision

- Product name is **Margin6**.
- Target: independent groups, **~2–10 locations**.
- **Internal-demo ready**; paused for trust/workflow repair — **not commercial-ready**.
- Core workflow: Count → Approve → Order → Invoice → Receipt → Dashboard/Alerts.
- Explicit **non-goals:** POS, recipes, theoretical food cost, payroll, scheduling, full accounting, trusted Food Cost/P&L/Money Lost for current pilot.

## Consequences

- New features outside scope require explicit founder approval.
- README and agents must not claim commercial readiness.
- Phase 4 dashboard splits deferred until trust epics complete.

## Not in this PR

No application code or schema changes.
